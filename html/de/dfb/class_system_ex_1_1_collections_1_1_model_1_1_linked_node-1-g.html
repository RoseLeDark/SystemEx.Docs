Detailed Description

LinkedNode<T> extends GenericNode<T> with bidirectional connectivity through Prev and Next pointers. This makes the type suitable for intrusive list structures that require constant‑time insertion, removal, splicing, and node‑level manipulation without auxiliary container objects.

The class provides a rich set of C++‑style operations commonly found in std::list and other intrusive list implementations:

  • InsertBefore() / InsertAfter() – insert an existing node directly before or after this node.
  • EmplaceBefore() / EmplaceAfter() – construct and insert a new node relative to this node.
  • Erase(), EraseNext(), ErasePrev() – remove this node or adjacent nodes from the chain.
  • Remove() / Detach() – detach this node from the list without destroying its value.
  • ReplaceWith() – replace this node with another node while preserving positional relationships.
  • SwapWith(), SwapNext(), SwapPrev() – exchange node positions without modifying stored values.
  • SpliceBefore() / SpliceAfter() – move a node from any list and insert it before or after this node.
  • Splice(first, last) – splice an entire node range into the position of this node.
  • Back() / Root() – navigate to the last or first node in the chain.
  • Advance() – move forward or backward by an offset.
  • AsRange(), Slice(), AsChain() – create STL‑style iterator ranges, slices, and chained ranges.

Unlike container‑based lists, LinkedNode<T> stores linkage directly inside each node. This intrusive design eliminates allocation overhead, improves cache locality, and enables fine‑grained structural manipulation without requiring a separate list object.

For forward‑only intrusive structures, use Node<T>.
For tree‑based intrusive structures, use TreeNode<T>.
