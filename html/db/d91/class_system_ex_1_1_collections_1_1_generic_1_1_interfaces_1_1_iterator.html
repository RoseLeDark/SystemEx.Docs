The iterator is cloned before advancing.

In C# iterators are reference‑based objects; advancing the original iterator would mutate the caller's iterator state.

Cloning ensures that Advance behaves like the C++ STL version: it returns a new iterator positioned n steps ahead, while the original iterator remains unchanged.

static

The iterator is cloned before advancing.

In C# iterators are objects, not value‑types.
Without cloning, advancing would mutate the caller's iterator, breaking STL‑style semantics and making algorithms like Distance, LowerBound, UpperBound or hashing routines unsafe.

Cloning preserves the expected C++ behavior:
Advance returns a new iterator at first + n, leaving first untouched.

This is the STL‑style equivalent of std::prev.
Internally delegates to Advance<T>(IRandomAccessIterator<T>, long) with a negative offset.

Because Advance supports both forward and backward stepping, this method performs actual backward movement using IRandomAccessIterator<T>.Back.

The original iterator remains unchanged.
